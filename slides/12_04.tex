\begin{frame}{Universal relations and Equivalence Relations}
\protect\hypertarget{universal-relations-and-equivalence-relations}{}
Strictly speaking, S5 is the logic of frames where R is an
\textbf{equivalence relation}. That is, it is a relation that is:

\begin{itemize}
\tightlist
\item
  Transitive
\item
  Reflexive
\item
  Symmetric
\end{itemize}

But we can simplify things by just focussing on the case where R is a
\textbf{universal} relation - that is, a relation that holds between any
two points.
\end{frame}

\begin{frame}{Universal relations and Equivalence Relations}
\protect\hypertarget{universal-relations-and-equivalence-relations-1}{}
Not all equivalence relations are universal relations:

\begin{itemize}
\tightlist
\item
  The relation \emph{Has the same birthday as} is an equivalence
  relation, but not universal.
\end{itemize}

But anything that's true on all models for universal R will be true on
all models for equivalence relation R.

\begin{itemize}
\tightlist
\item
  That's because from the perspective of any point, R may as well be
  universal.
\item
  All that matters to truth at any point is what happens at points you
  can reach in a finite number of steps.
\item
  So R may as well be a universal relation.
\end{itemize}
\end{frame}

\begin{frame}{Simple Rules for S5}
\protect\hypertarget{simple-rules-for-s5}{}
Drop the rules you've already got for \(\Box\) and \(\Diamond\). And
drop the decimal points; all worlds just get referred to by a single
number. \pause 

\begin{itemize}
\tightlist
\item
  If \(\Box A\) is true at \(x\), then for any \(y\) on the branch,
  \(A\) is true at \(y\).
\item
  If \(\Diamond A\) is false at \(x\), then for any \(y\) on the branch,
  \(A\) is false at \(y \pause\).
\item
  If \(\Box A\) is false at \(x\), then add a new world \(y\) with \(A\)
  false at \(y\).
\item
  If \(\Diamond A\) is true at \(x\) then add a new world \(y\) with
  \(A\) true at \(y\).
\end{itemize}
\end{frame}
