\begin{frame}{A Proof}
\protect\hypertarget{a-proof-1}{}
\begin{itemize}
\tightlist
\item
  A proof ends when you get to the sentence you were aiming for.
\item
  This is different to tables, where you have to fill in everything.
\item
  And it is different to trees, where you have to either close all
  branches, or tick all lines.
\item
  Instead, proofs end when they have done their job.
\item
  In most cases, you could continue; you just choose not to.
\end{itemize}
\end{frame}

\begin{frame}{Get To?}
\protect\hypertarget{get-to}{}
\begin{itemize}
\tightlist
\item
  How do we `get' from premises to where we want to go?
\item
  There are rules that allow us to generate new lines from old lines.
\item
  The premises are lines we can use for this purpose, but they aren't
  the only ones.
\item
  Indeed, once the proof starts, that horizontal line carving off
  premises from conclusions does very little work.
\end{itemize}
\end{frame}

\begin{frame}{Rules}
\protect\hypertarget{rules}{}
\begin{itemize}
\tightlist
\item
  Mostly, the rules come in pairs, two for each connective.
\item
  They will be called \textbf{introduction} and \textbf{elimination}
  rules.
\item
  \emph{Very} roughly, the introduction rules are for when the
  connective appears in the desired conclusion, and the elimination
  rules are for when the connective appears in a premise.
\item
  But this is only rough - the rules are just rules.
\end{itemize}
\end{frame}

\begin{frame}{Reiteration}
\protect\hypertarget{reiteration}{}
\begin{itemize}
\tightlist
\item
  The first rule the book introduces, in 16.2, is one of the rare rules
  that is not an introduction or elimination rule.
\item
  It says that you are allowed to repeat yourself.
\item
  If a line appears somewhere, it can get repeated later.
\end{itemize}
\end{frame}

\begin{frame}{Why Reiteration}
\protect\hypertarget{why-reiteration}{}
\begin{itemize}
\tightlist
\item
  It turns out it has some technical uses.
\item
  But really it is mostly so we can make the arguments sound more like
  real life.
\item
  In real arguments, if you argue for \emph{p}, then three paragraphs
  later argue for \emph{if p, q}, you might remind the audience that
  \emph{p}, rather than just immediately infer \emph{q}.
\item
  That reminder is just reiteration.
\end{itemize}
\end{frame}
